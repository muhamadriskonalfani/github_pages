Database

Definisi

Database adalah kumpulan informasi yang disimpan didalam media penyimpanan secara sistematik sehingga dapat diperiksa menggunakan suatu program untuk memperoleh informasi dari basis data tersebut.

Database Meliputi
  • DBMS
  • Entitas / tabel
  • Atribut
  • Tupple / Record
  • Data value
  • Kunci / Key
  • Relasi
Penjelasan

  • DBMS :
    Database Management System adalah aplikasi yang digunakan untuk mengoperasikan sebuah database. Contohnya MySQL, PostgreSQL, Oracle Database, dan sebagainya.

  • Entitas :
    Entitas juga bisa disebut sebagai nama table yang berada pada database.

  • Atribut :
    Atribut berada dalam sebuah entitas atau tabel, contoh tabel 'tb_mahasiswa' memiliki artibut 'id_mahasiswa', 'nama_mahasiswa', dan sebagainya.

  • Record :
    Record adalah rekaman suatu data yang meliputi semua atribut pada sebuah identitas. Record adalah semua data value dalam satu baris pada tabel.

  • Data Value :
    Data value adalah isi dari sebuah kolom pada tabel.

  • Key :
    Kunci atau Key adalah data value yang membedakan antara satu record dengan record yang lain, ada 2 jenis key yaitu.

    • PRIMARY KEY :
      Adalah kunci yang digunakan sebagai pembeda suatu record dengan record yang lain.

    • FOREIGN KEY :
      Adalah kunci yang digunakan untuk menghubungkan atribut 1 dengan atribut PRIMARY KEY di tabel lain.

  • Relasi :
    Relasi adalah hubungan antar entitas atau tabel pada sebuah database.

DBMS MySQL

Definisi

MySQL adalah sistem manajemen basis data relasional (RDBMS) yang populer dan sering digunakan untuk menyimpan, mengelola, dan mengambil data. Di dalam MySQL, ada yang dinamakan sebagai query. Query MySQL adalah perintah-perintah atau pernyataan yang digunakan untuk berinteraksi dengan basis data MySQL.

Query MySQL

Berikut beberapa perintah dasar yang harus diketahui sebelum mengoperasikan sebuah database pada MySQL.

  • Login :
    Hal pertama sebelum mengelola sebuah database adalah melakukan login. Anda bisa login sebagai user ataupun sebagai root.

    mysql -u username -p

  • Logout :
    Anda juga bisa melakukan logout untuk keluar dari user atau root.

    exit

  • Menampilkan semua database :
    Setelah login, Anda bisa menampilkan semua database dari user tersebut.

    SHOW DATABASES;

  • Memilih sebuah database :
    Sebelum mengelola sebuah database, Anda harus memilih database mana yang ingin Anda kelola.

    USE dbname;

  • Menampilkan semua tabel :
    Setelah memilih database, Anda bisa menampilkan semua tabel yang ada pada database tersebut.

    SHOW TABLES;

  • Menampilkan deskripsi tabel :
    Digunakan untuk menampilkan deskripsi dari sebuah tabel seperti atribut, tipe data, kunci, dan sebagainya.

    DESC tbname;

  • Menampilkan komposisi tabel :
    Digunakan untuk menampilkan perintah bagaimana sebuah tabel dibuat.

    SHOW CREATE TABLE tbname

Query MySQL

Definisi

Query adalah syntax atau perintah yang digunakan untuk mengakses dan menampilkan data pada sistem database. Query juga digunakan untuk membuat dan menghapus database atau tabel, menambahkan data value atau record, memberikan hak akses kepada user, membuat data saling berinteraksi, dan masih banyak lagi.

Macam-Macam Query SQL
  • DDL atau data definition language
  • DML atau data manipulation language
  • DCL atau data control language
DDL atau Data Definition Language

DDL atau Data Definition Language digunakan untuk mendefinisikan data pada database. Definisi lain, DDL digunakan untuk mengelola database, tabel, ataupun atribut pada tabel. Tidak sampai pada data value atau record.

Query pada DDL

  • CREATE :
    digunakan untuk membuat database maupun tabel

  • ALTER :
    digunakan untuk merubah stuktur tabel

  • DROP :
    digunakan untuk menghapus database maupun tabel

DML atau Data Manipulation Language

DML atau Data Manipulation Language digunakan untuk memanipulasi data pada tabel yang telah dibuat. Definisi lain, DML digunakan untuk mengelola data value atau record. Seperti menampilkan, menambah, mengubah, dan menghapus data.

Query pada DML

  • SELECT :
    digunakan untuk menampilkan data value pada tabel

  • INSERT :
    digunakan untuk mengisi data value pada tabel

  • UPDATE :
    digunakan untuk mengubah data value pada tabel

  • DELETE :
    digunakan untuk menghapus data value pada tabel

DCL atau Data Control Language

DCL atau Data Control Language adalah bagian dari bahasa SQL yang digunakan untuk mengelola hak akses dan izin pengguna dalam sebuah basis data.

Query pada DCL

  • GRANT :
    digunakan untuk mengizinkan user mengakses tabel dalam database

  • REVOKE :
    digunakan untuk membatalkan izin hak akses user

  • COMMIT :
    digunakan untuk menetapkan penyimpanan database

  • ROLLBACK :
    digunakan untuk membatalkan penyimpanan database

DDL Atau Data Definition Language

Definisi
DDL atau Data Definition Language digunakan untuk mendefinisikan data pada database. Definisi lain, DDL digunakan untuk mengelola database, tabel, ataupun atribut pada tabel. Tidak sampai pada data value atau record.
DDL Meliputi

CREATE

CREATE
digunakan untuk membuat database atau tabel. contoh membuat database dengan nama 'db_kampus' dan tabel dengan nama 'tb_mahasiswa'.

baca selengkapnya

  • Membuat database baru

    CREATE DATABASE db_kampus;

  • Membuat tabel baru

    CREATE TABLE tb_mahasiswa;

  • Membuat database atau tabel dengan 'IF NOT EXIST'

    CREATE DATABASE IF NOT EXIST db_kampus;
IF NOT EXIST digunakan untuk mengecek apakah ada database atau tabel dengan nama tersebut. jika ada maka tidak dibuat dan jika tidak ada maka dibuat
ALTER

ALTER
digunakan untuk mengganti nama tabel dan memanipulasi atribut tabel. contoh merubah nama tabel, menambah field, merubah tipe data, merubah nama field, dan menghapus field.

baca selengkapnya

  • Merubah nama tabel

    ALTER TABLE mahasiswa RENAME tb_mahasiswa;

  • Menambah field pada tabel

    ALTER TABLE mahasiswa ADD prodi VARCHAR(20);

  • Merubah tipe data 

    ALTER TABLE mahasiswa MODIFY prodi VARCHAR(30);

  • Merubah nama field

    ALTER TABLE mahasiswa CHANGE nama_lama nama_baru VARCHAR(20);

  • Menghapus field pada tabel

    ALTER TABLE mahasiswa DROP prodi;

DROP
digunakan untuk menghapus database maupun tabel. contoh database dengan nama 'db_kampus' dan tabel dengan nama 'tb_mahasiswa'.

baca selengkapnya

  • Menghapus database

    DROP DATABASE db_kampus;

  • Menghapus tabel

    DROP TABLE tb_mahasiswa;

  • Menghapus database atau tabel dengan 'IF EXIST'

    DROP DATABASE IF EXIST db_kampus;
IF EXIST digunakan untuk mengecek apakah ada database atau tabel dengan nama tersebut. jika ada maka akan dihapus

DML Atau Data Manipulation Language

Definisi
DML atau Data Manipulation Language digunakan untuk memanipulasi data pada tabel yang telah dibuat. Definisi lain, DML digunakan untuk mengelola data value atau record. Seperti menampilkan, menambah, mengubah, dan menghapus data.
DML Meliputi
  • SELECT
  • INSERT
  • UPDATE
  • DELETE
SELECT

SELECT
digunakan untuk menampilkan data value pada tabel. bisa menampilkan satu, beberapa, atau bahkan semua data value sekaligus.

baca selengkapnya

  • Menampilkan semua data value. tanda * artinya tampilkan semua

    SELECT * FROM mahasiswa;

  • Menampilkan semua nama_mahasiswa

    SELECT nama_mahasiswa FROM tb_mahasiswa;

  • Menampilkan semua nama_mahasiswa dan jenis_kelamin

    SELECT nama_mahasiswa, jenis_kelamin FROM tb_mahasiswa;

  • Menampilkan karakteristik apa saja yang ada di sebuah kolom dari tb_mahasiswa

    SELECT DISTINCT jenis_kelamin FROM tb_mahasiswa;

  • Menampilkan semua mahasiswa dengan jenis kelamin laki-laki

    SELECT * FROM tb_mahasiswa WHERE jenis_kelamin = "laki-laki";

  • Menampilkan semua mahasiswa dengan usia antara 20 dan 22

    SELECT * FROM tb_mahasiswa WHERE usia BETWEEN 20 AND 22;

  • Menampilkan semua nama mahasiswa dengan akhiran 'an'

    SELECT * FROM tb_mahasiswa WHERE nama_mahasiswa LIKE "%an";
INSERT

INSERT
digunakan untuk memasukan record baru ke dalam tabel. bisa satu record atau bahkan banyak record sekaligus.

baca selengkapnya

  • Menambah satu record

    INSERT INTO mahasiswa VALUES ('20121092', 'Budi', 'Informatika');

  • Menambah dua atau lebih record

    INSERT INTO mahasiswa VALUES ('20121092', 'Budi', 'Informatika'), ('20121093', 'Dita', 'Informatika'), ('20121094', 'Andi', 'Informatika');
Ketika memasukan data, usahakan data tersebut sesuai dengan urutan fieldnya. perhatikan nama, tipe data, dan jumlah karakternya
UPDATE

UPDATE
digunakan untuk mengganti data lama dengan data baru menggunakan primary key. Bisa mengganti satu data atau bahkan banyak data sekaligus.

baca selengkapnya

  • Merubah data value prodi dari 'DKV' dirubah menjadi 'TI'

    UPDATE mahasiswa SET prodi = 'TI' WHERE nim = 20121092;

  • Merubah data value prodi dari 'DKV' dirubah menjadi 'TI', dan nama mahasiswa dari 'alexander' dirubah menjadi 'bobby'

    UPDATE mahasiswa SET prodi = 'TI', nama_mahasiswa = 'booby' WHERE nim = 20121092;
Penting untuk memberikan primary key karena jika tidak maka semua data pada tabel akan berubah
DELETE

DELETE
digunakan untuk menghapus data value pada tabel. bisa menghapus satu, beberapa, atau bahkan semua data value sekaligus.

baca selengkapnya

  • Menghapus satu record pada tabel

    DELETE FROM mahasiswa WHERE nim = 20121092;

  • Menghapus semua record mahasiswa yang jenis kelaminya laki-laki

    DELETE FROM mahasiswa WHERE jenis_kelamin = 'laki-laki';

  • Menghapus semua record pada tabel

    DELETE FROM mahasiswa;
Penting untuk memberikan nama bagian tabel yang akan dihapus karena jika tidak maka semua data value pada tabel akan dihapus

DCL Atau Data Control Language

Definisi

DCL atau Data Control Language, adalah bagian dari bahasa SQL yang digunakan untuk mengelola hak akses dan izin pengguna dalam sebuah basis data. DCL digunakan untuk mengatur perizinan seperti GRANT (memberikan hak akses kepada user lain) dan REVOKE (mencabut hak akses yang telah diberikan) terhadap objek-objek dalam basis data.

GRANT

Perintah ini digunakan untuk memberikan izin akses kepada pengguna atau peran tertentu pada objek-objek dalam basis data. Izin-izin ini dapat mencakup hak untuk melakukan operasi tertentu seperti SELECT, INSERT, UPDATE, DELETE, dll.

baca selengkapnya

Penulisan GRANT
GRANT tipe_izin ON nama_database.nama_tabel TO 'nama_user'@'localhost';
GRANT dilakukan di dalam root. pastikan untuk membuat user terlebih dahulu. untuk mengecek apakah sudah berhasil, bisa keluar dari root dan login sebagai user

Berikut beberapa contoh pemberian hak akses kepada user

  • Memberikan satu hak akses, contoh hak akses CREATE

    GRANT CREATE ON nama_database.nama_tabel TO 'nama_user'@'localhost';

  • Memberikan dua atau lebih hak akses, contoh hak akses CREATE dan DROP

    GRANT CREATE, DROP ON nama_database.nama_tabel TO 'nama_user'@'localhost';

  • Memberikan semua hak akses

    GRANT ALL ON nama_database.nama_tabel TO 'nama_user'@'localhost';
REVOKE

Perintah ini digunakan untuk mencabut izin akses yang telah diberikan sebelumnya kepada pengguna atau peran pada objek-objek dalam basis data.

baca selengkapnya

Penulisan REVOKE
REVOKE ALL ON nama_database.nama_tabel FROM 'nama_user'@'localhost';
REVOKE dilakukan di dalam root. tentukan tipe izin, database, tabel, dan user sebelum melakukan REVOKE

Berikut beberapa cara mencabut atau menarik kembali hak akses dari user

  • Menarik satu hak akses, contoh menarik akses CREATE

    REVOKE CREATE ON nama_database.nama_tabel FROM 'nama_user'@'localhost';

  • Menarik dua atau lebih hak akses, contoh menarik akses CREATE dan DROP

    REVOKE CREATE, DROP ON nama_database.nama_tabel FROM 'nama_user'@'localhost';

  • Menarik semua hak akses

    REVOKE ALL ON nama_database.nama_tabel FROM 'nama_user'@'localhost';
COMMIT

COMMIT adalah pernyataan yang digunakan untuk mengakhiri transaksi yang sedang berjalan dan secara permanen menyimpan semua perubahan yang telah dilakukan dalam transaksi tersebut ke dalam basis data.

baca selengkapnya
Contoh penggunaan perintah COMMIT

Misalnya, Saya ingin menambahkan sebuah data ke dalam tabel yang saya buat.

  • Pertama, ketikan perintah 'START TRANSACTION'

    START TRANSACTION;

  • Setelah itu masukan data value

    INSERT INTO nama_tabel VALUES ('1', 'Alexander', 'laki-laki', 'jawa tengah');

  • Selanjutnya ketikan perintah 'COMMIT' untuk menyimpan perubahan yang terjadi

    COMMIT;

Dengan menggunakan 'transaction' saat melakukan perintah insert, update, delete maka transaksi sebenarnya belum dilakukan secara permanen. Artinya operasi tersebut masih bisa di rollback atau dibatalkan.

Contoh lain penggunaan COMMIT

    BEGIN;  -- Mulai transaksi
    UPDATE accounts SET balance = balance - 100 WHERE account_id = 1;
    UPDATE accounts SET balance = balance + 100 WHERE account_id = 2;
    COMMIT; -- Akhiri transaksi dan simpan perubahan

Dalam contoh di atas, pernyataan COMMIT mengonfirmasi bahwa kedua pembaruan saldo berhasil dilakukan dan perubahan tersebut disimpan secara permanen ke dalam basis data.

ROLLBACK

ROLLBACK adalah pernyataan yang digunakan untuk membatalkan semua perubahan yang terjadi selama transaksi yang sedang berjalan. Ketika terjadi kesalahan dan Anda ingin mengembalikan basis data ke keadaan sebelum transaksi dimulai, Anda dapat menggunakan pernyataan ROLLBACK.

baca selengkapnya
Contoh penggunaan perintah ROLLBACK

Misalnya, Saya menghapus data yang salah dan Saya ingin mengembalikan data tersebut

  • Pertama, ketikan perintah 'START TRANSACTION'

    START TRANSACTION;

  • Terjadi kesalahan menghapus data

    DELETE FROM nama_tabel WHERE nim = 1;

  • Selanjutnya ketikan perintah ROLLBACK untuk mengembalikan data

    ROLLBACK;

  • Data yang dihapus berhasil dikembalikan seperti semula

Perintah rollback digunakan untuk mengembalikan data pada database ke bentuk awal. Perintah rollback sama fungsinya dengan perintah undo atau ctrl + Z pada keyboard, jadi apabila ada data yang dihapus maka bisa dikembalikan seperti semula dengan menggunakan perintah rollback.

Contoh lain penggunaan ROLLBACK

    BEGIN;  -- Mulai transaksi
    UPDATE inventory SET stock = stock - 10 WHERE product_id = 123;
    UPDATE orders SET status = 'completed' WHERE order_id = 456;
    -- Kesalahan terjadi di sini
    ROLLBACK; -- Batalkan semua perubahan dalam transaksi

Dalam contoh di atas, karena terjadi kesalahan setelah pembaruan inventaris tetapi sebelum pembaruan pesanan, pernyataan ROLLBACK akan membatalkan pembaruan inventaris dan mengembalikan basis data ke keadaan sebelum transaksi dimulai.

  • DELETE :
    Memberikan akses menghapus record di tabel

User

Pengguna (user) dalam konteks basis data merujuk pada individu atau entitas yang memiliki akses ke basis data dan diizinkan untuk melakukan operasi seperti query dan manipulasi data tertentu sesuai dengan hak akses yang diberikan.

baca selengkapnya

Berikut langkah-langkah untuk membuat user.

  • Login ke MySQL :
    Pertama, buka terminal dan login ke server MySQL dengan hak akses administratif (biasanya dengan akun root).

    mysql -u root -p

  • Buat User baru :
    Gunakan perintah 'CREATE USER' untuk membuat user baru. Gantilah 'username' dengan nama pengguna yang Anda ingin buat, dan 'password' dengan kata sandi yang akan digunakan oleh pengguna ini.

    CREATE USER 'nama_user'@'localhost' IDENTIFIED BY 'password_user';

    Tidak boleh ada spasi diantara simbol '@'.

  • Berikan Hak Akses :
    Setelah membuat user, Anda perlu memberikan hak akses tertentu ke pengguna tersebut. Gunakan perintah 'GRANT' untuk memberikan izin-izin yang diinginkan.

    GRANT tipe_izin ON dbname.* TO 'username'@'localhost';

  • Terapkan Perubahan :
    Setelah memberikan izin-izin, jangan lupa untuk menerapkan perubahan dengan perintah 'FLUSH PRIVILEGES'.

    FLUSH PRIVILEGES;

    Jika Anda tidak menjalankan 'FLUSH PRIVILEGES', pengguna mungkin tidak memiliki akses yang diharapkan atau perubahan izin yang Anda lakukan mungkin tidak akan berlaku.

Export Dan Import MySQL

Definisi

Ekspor dan impor database adalah proses untuk menyimpan data dari suatu sistem basis data ke dalam file, lalu mengambil data tersebut dari file tersebut dan memasukkannya ke dalam sistem basis data lainnya. Ini berguna saat Anda ingin berbagi data antara berbagai sistem, mencadangkan data, atau mengganti sistem basis data.

Ekspor Database

Ekspor database adalah proses mencadangkan data dari sistem basis data tertentu ke dalam bentuk file eksternal dan nantinya dapat di impor kembali ke sistem basis data yang sama atau berbeda.

baca selengkapnya

Berikut langkah-langkah ekspor database maupun tabel.

  • Pilih Database :
    Pilih database yang ingin Anda ekspor.

  • Pilih Tabel :
    Jika Anda hanya ingin meng-ekspor bagian tertentu dari database pilih tabel-tabel yang ingin Anda ekspor.

  • Ekspor :
    Gunakan perintah 'mysqldump' untuk ekspor data.

    Ekspor database 

    mysqldump -u username -p dbname > backup.sql

    Ekspor table 

    mysqldump -u username -p dbname tbname ... > backup.sql

    • Ganti

      'username'
      dengan nama user MySQL Anda.

    • Ganti

      'dbname'
      dengan nama database yang ingin anda ekspor (untuk ekspor database).

    • Ganti

      'tbname'
      dengan nama tabel yang ingin anda ekspor (untuk ekspor tabel). Bisa lebih dari satu tabel.

    • 'backup.sql'
      adalah nama file tempat anda menyimpan database maupun tabel yang di ekspor.

    • Anda akan diminta untuk memasukkan kata sandi MySQL setelah menjalankan perintah ini.

  • Simpan File :
    File backup.sql akan berisi database atau tabel yang nantinya dapat di impor kembali ke sistem basis data yang sama atau berbeda.

Penting untuk mengingat dimana anda menyimpan file 'backup.sql' ini karena nantinya akan digunakan untuk proses impor. Anda bisa membuat folder baru dengan nama 'ekspor' untuk menyimpan file-file backup tersebut.

Impor Database

Impor database adalah proses memasukkan data yang sebelumnya diekspor dari sistem basis data lain ke dalam sistem basis data yang sedang Anda gunakan.

baca selengkapnya

Berikut langkah-langkah impor database maupun tabel.

  • Pilih Database :
    Pilih database yang anda inginkan jika anda ingin meng-impor tabel (database yang sudah ada sebelumnya).

  • Buat Database :
    Buat database baru jika anda ingin meng-impor database.

  • Cari file yang ingin di impor :
    Masuk ke direktori 'ekspor' yang sebelumnya dibuat.

    cd c/ekspor/

  • Pilih file yang ingin di impor :
    Setelah masuk ke direktori 'ekspor' anda bisa cek di dalamnya apakah ada file yang ingin anda impor.

    dir

  • Impor :
    Gunakan perintah SQL yang sesuai untuk mengimpor data dari file.

    mysql -u username -p dbname < backup.sql

    • Ganti

      'username'
      dengan nama user MySQL Anda.

    • Ganti

      'dbname'
      dengan nama database tempat anda akan menyimpan database maupun tabel yang di impor.

    • 'backup.sql'
      adalah nama file tempat database maupun tabel yang di ekspor sebelumnya untuk di impor.

    • Anda akan diminta untuk memasukkan kata sandi MySQL setelah menjalankan perintah ini.

  • Verifikasi :
    Pastikan bahwa data telah diimpor dengan benar dengan memeriksa tabel dan rekaman.

Primary Key

Definisi

Primary key digunakan untuk membedakan antara identitas satu dengan identitas yang lainya dalam sebuah tabel atau entitas. Untuk value atribut primary key tidak boleh kosong atau null dan satu tabel hanya punya satu primary key.

Membuat Primary Key

Berikut adalah beberapa cara untuk membuat atribut PRIMARY KEY. Contohnya membuat atribut PRIMARY KEY dengan nama 'id_mahasiswa' pada tabel 'tb_mahasiswa'

baca selengkapnya

  • Membuat atribut PRIMARY KEY saat membuat tabel

    CREATE TABLE tb_mahasiswa ( id_mahasiswa INT(10) NOT NULL PRIMARY KEY );

  • Menambahkan atribut PRIMARY KEY pada tabel

    ALTER TABLE tb_mahasiswa ADD id_mahasiswa INT(10) NOT NULL PRIMARY KEY FIRST;

  • Mengubah atribut biasa menjadi atribut PRIMARY KEY

    ALTER TABLE tb_mahasiswa CHANGE id_mahasiswa id_mahasiswa INT(10) NOT NULL PRIMARY KEY FIRST;

  • Menambahkan PRIMARY KEY pada atribut biasa

    ALTER TABLE tb_mahasiswa ADD PRIMARY KEY (id_mahasiswa);

  • Menghapus PRIMARY KEY pada atribut

    ALTER TABLE tb_mahasiswa DROP PRIMARY KEY;
Increment dan Decrement

baca selengkapnya

Reset Auto Increment

AUTO INCREMENT membuat suatu atribut memiliki data dengan nomor urut, namun saat ada data yang dihapus maka akan ada nomor urut yang hilang, dengan itu untuk membuat nomor manjadi urut kembali maka bisa dilakukan dengan cara berikut

baca selengkapnya

Foreign Key

Definisi

FOREIGN KEY adalah kunci asing pada sebuah atribut dalam sebuah tabel yang menghubungkan atribut satu dengan atribut lain pada tabel yang berbeda. FOREIGN KEY digunakan untuk membangun hubungan antara dua tabel dalam sebuah basis data.

Membuat Foreign Key

Misalkan ada dua tabel 'Pelanggan' dan 'Pesanan'. Setiap pesanan harus dikaitkan dengan seorang pelanggan. Anda dapat membuat atribut 'id_pelanggan' dalam tabel 'Pesanan' sebagai FOREIGN KEY yang merujuk pada atribut 'id_pelanggan' dalam tabel 'Pelanggan'. Jadi setiap pesanan terkait dengan pelanggan tertentu dalam database.

baca selengkapnya

Berikut langkah-langkah untuk membuat FOREIGN KEY

  • Buat dua tabel :
    Contoh 'tb_pelanggan' dan 'tb_pesanan'.

    
    CREATE TABLE tb_pelanggan (
        id_pelanggan VARCHAR(10) NOT NULL PRIMARY KEY   
    );
                                        
    
    CREATE TABLE tb_pesanan (
        id_pesanan VARCHAR(10) NOT NULL PRIMARY KEY,    
        id_pelanggan VARCHAR(10) NOT NULL
    );

  • Buat FOREIGN KEY :
    Hubungkan 'id_pelanggan' pada 'tb_pesanan' ke 'id_pelanggan' yang ada di 'tb_pelanggan'.

    ALTER TABLE tb_pesanan ADD FOREIGN KEY (id_pelanggan) REFERENCES tb_pelanggan (id_pelanggan);

  • Deskripsikan Tabel :
    Deskripsikan tabel untuk melihat hasilnya.

    DESC tb_pesanan;

  • FOREIGN KEY berhasil dibuat :
    Sekarang atribut 'id_pelanggan' pada tabel 'tb_pesanan' sudah menjadi FOREIGN KEY.

Menghapus Foreign Key

Selanjutnya adalah langkah-langkah menghapus FOREIGN KEY. Menghapus FOREIGN KEY dapat dilakukan apabila FOREIGN KEY yang dibuat salah atau tidak lagi dibutuhkan.

baca selengkapnya

Berikut langkah-langkah untuk menghapus FOREIGN KEY

  • Tampilkan komponen dari tabel 'tb_pesanan' :
    Kita butuh untuk mengetahui CONSTRAINT dari atribut FOREIGN KEY-nya.

    SHOW CREATE TABLE tb_pesanan;

  • Hapus CONSTRAINT :
    Hapus CONSTRAINT dari atribut FOREIGN KEY

    ALTER TABLE tb_pesanan DROP FOREIGN KEY kode_constraint_tabel;

  • Tampilkan komponen tabel 'tb_pesanan' :
    Yang tadinya ada CONSTRAINT berubah menjadi KEY, KEY ini harus dihapus.

    SHOW CREATE TABLE tb_pesanan;

  • Hapus KEY :
    Selanjutnya hapus KEY dari 'id_pelanggan'.

    ALTER TABLE tb_pesanan DROP KEY id_pelanggan;

  • Hapus FOREIGN KEY berhasil :
    Sekarang atribut 'id_pelanggan' pada tabel 'tb_pesanan' bukan lagi menjadi FOREIGN KEY.

On Delete Cascade

Definisi

On DELETE CASCADE adalah sebuah fitur yang ada dalam basis data yang digunakan untuk menjaga integritas referensial antara tabel-tabel dalam basis data relasional. Integritas referensial adalah prinsip yang memastikan bahwa hubungan antara data dalam tabel-tabel terjaga dengan benar.

ON DELETE CASCADE

Berikut adalah contoh sederhana kode SQL untuk membuat dua tabel yang memiliki hubungan referensial dengan pengaturan On DELETE CASCADE

baca selengkapnya

    -- Membuat tabel 'tb_pesanan'
    CREATE TABLE tb_pesanan (
        id_pesanan INT PRIMARY KEY,
        nama_pelanggan VARCHAR(50)
    );
    
    -- Membuat tabel 'tb_produk_pesanan' dengan hubungan referensial ke tabel 'tb_pesanan'
    CREATE TABLE tb_produk_pesanan (
        id_produk INT PRIMARY KEY,
        id_pesanan INT,
        nama_produk VARCHAR(50),
        quantity INT,
        FOREIGN KEY (id_pesanan) REFERENCES tb_pesanan(id_pesanan) ON DELETE CASCADE
    );
  • Tabel 'tb_pesanan' mempunyai atribut 'id_pesanan' sebagai PRIMARY KEY.
  • Tabel 'tb_produk_pesanan' mempunyai atribut 'id_produk' sebagai PRIMARY KEY dan 'id_pesanan' sebagai FOREIGN KEY yang merujuk pada tabel 'tb_pesanan'.
  • Pengaturan ON DELETE CASCADE diterapkan pada kunci asing 'id_pesanan' di tabel 'tb_produk_pesanan'
  • Jadi jika sebuah pesanan dihapus dari tabel 'tb_pesanan', semua data produk pesanan yang terkait akan dihapus secara otomatis dari tabel 'tb_produk_pesanan'.
Implementasi

Ada dua buah tabel yaitu 'tb_pesanan' dan 'tb_produk_pesanan', jika sebuah pesanan dihapus dari tabel 'tb_pesanan', maka semua data produk pesanan yang terkait akan dihapus secara otomatis dari tabel 'tb_produk_pesanan'.

Data Trigger

Definisi

Data trigger dalam basis data adalah mekanisme yang memungkinkan tindakan otomatis terjadi saat suatu peristiwa tertentu terjadi pada tabel.

Data trigger adalah bagian dari kode yang diaktifkan oleh suatu peristiwa tertentu, seperti pengisian data, pembaruan data, atau penghapusan data. Ketika peristiwa ini terjadi, trigger akan dieksekusi dan menjalankan serangkaian tindakan yang telah ditentukan sebelumnya.

Penggunaan Data Trigger

1. Validasi Data :
Anda dapat membuat trigger yang memeriksa apakah data yang dimasukkan ke dalam tabel memenuhi persyaratan atau tidak. Misalnya, trigger untuk memastikan bahwa nilai tanggal yang dimasukkan adalah di masa depan, atau bahwa sebuah nilai tidak melebihi batas dari yang ditentukan.


    DELIMITER //
    CREATE TRIGGER validate_future_date
    BEFORE INSERT ON nama_tabel
    FOR EACH ROW
    BEGIN
        IF NEW.tanggal < CURDATE() THEN
            SIGNAL SQLSTATE '45000'
            SET MESSAGE_TEXT = 'Tanggal harus di masa depan';
        END IF;
    END;
    //
    DELIMITER ;

  • 'validate_future_date'
    adalah nama trigger yang Anda tentukan.

  • 'BEFORE INSERT ON nama_tabel'
    menunjukkan bahwa trigger akan diaktifkan sebelum data dimasukkan ke dalam tabel dengan nama 'nama_tabel'.

  • 'FOR EACH ROW'
    menunjukkan bahwa trigger akan dieksekusi untuk setiap baris yang dimasukkan.

  • 'NEW.tanggal'
    merujuk pada nilai yang akan dimasukkan ke dalam kolom 'tanggal'.

  • 'CURDATE()'
    mengembalikan tanggal saat ini.

  • 'SIGNAL SQLSTATE '45000''
    digunakan untuk memicu pengecualian dengan kode kesalahan khusus.

  • 'SET MESSAGE_TEXT = 'Tanggal harus di masa depan''
    menetapkan pesan kesalahan yang akan ditampilkan jika kondisi tidak terpenuhi.

2. Log Pembaruan :
Trigger juga dapat digunakan untuk mencatat setiap kali data dalam tabel diubah. Dengan cara ini, Anda dapat melacak siapa yang melakukan perubahan, kapan perubahan itu terjadi, dan apa yang diubah.

                                                
    DELIMITER //
    CREATE TRIGGER log_data_changes
    AFTER UPDATE ON nama_tabel
    FOR EACH ROW
    BEGIN
        INSERT INTO log_pembaruan (tabel, kolom, nilai_sebelum, nilai_setelah, waktu_pembaruan)
        VALUES ('nama_tabel', 'kolom_yang_diubah', OLD.kolom_yang_diubah, NEW.kolom_yang_diubah, NOW());
    END;
    //
    DELIMITER ;

  • 'log_data_changes'
    adalah nama trigger yang Anda tentukan.

  • 'AFTER UPDATE ON nama_tabel'
    menunjukkan bahwa trigger akan diaktifkan setelah data diperbarui pada tabel dengan nama nama_tabel.

  • 'FOR EACH ROW'
    menunjukkan bahwa trigger akan dieksekusi untuk setiap baris yang diperbarui.

  • 'INSERT INTO log_pembaruan'
    adalah perintah untuk memasukkan data pembaruan ke dalam tabel log_pembaruan.

  • 'OLD.kolom_yang_diubah' dan 'NEW.kolom_yang_diubah'
    adalah nilai sebelum dan sesudah dari kolom yang diubah.

  • 'NOW()'
    digunakan untuk mengambil waktu saat ini saat pembaruan terjadi.

3. Penghitungan Otomatis :
Trigger dapat digunakan untuk menghitung atau mengagregasi data secara otomatis saat ada perubahan dalam tabel yang relevan. Misalnya, trigger dapat menghitung jumlah produk yang tersedia setiap kali ada transaksi penjualan.

                                                
    DELIMITER //
    CREATE TRIGGER update_stock_count
    AFTER INSERT ON transaksi_penjualan
    FOR EACH ROW
    BEGIN
        DECLARE product_id INT;
        
        SET product_id = NEW.product_id;
        
        UPDATE stok_produk
        SET jumlah = jumlah - 1
        WHERE id = product_id;
    END;
    //
    DELIMITER ;

  • 'update_stock_count'
    adalah nama trigger yang Anda tentukan.

  • 'AFTER INSERT ON transaksi_penjualan'
    menunjukkan bahwa trigger akan diaktifkan setelah data dimasukkan ke dalam tabel transaksi_penjualan.

  • 'FOR EACH ROW'
    menunjukkan bahwa trigger akan dieksekusi untuk setiap baris yang dimasukkan.

  • 'DECLARE product_id INT;'
    mendeklarasikan variabel product_id yang akan digunakan dalam trigger.

  • 'SET product_id = NEW.product_id;'
    mengatur nilai product_id dengan nilai dari kolom product_id dalam baris yang dimasukkan.

  • 'UPDATE stok_produk'
    adalah perintah untuk memperbarui tabel stok produk.

  • 'SET jumlah = jumlah - 1'
    mengurangkan jumlah stok produk.

  • 'WHERE id = product_id;'
    mengatur kondisi di mana pembaruan stok hanya dilakukan untuk produk dengan id yang sesuai dengan product_id yang baru saja dimasukkan.

4. Pemberitahuan dan Notifikasi :
Anda bisa menggunakan trigger untuk mengirimkan pemberitahuan atau notifikasi ke sistem lain atau pengguna ketika suatu peristiwa penting terjadi, seperti ketika stok suatu produk hampir habis.

                                            
    DELIMITER //
    CREATE TRIGGER low_stock_notification
    AFTER UPDATE ON stok_produk
    FOR EACH ROW
    BEGIN
        IF NEW.jumlah <= 10 THEN
            INSERT INTO notifikasi (pesan, waktu)
            VALUES ('Stok produk dengan ID ' + NEW.id + ' hampir habis', NOW());
        END IF;
    END;
    //
    DELIMITER ;

  • 'low_stock_notification'
    adalah nama trigger yang Anda tentukan.

  • 'AFTER UPDATE ON stok_produk'
    menunjukkan bahwa trigger akan diaktifkan setelah data diperbarui pada tabel stok_produk.

  • 'FOR EACH ROW'
    menunjukkan bahwa trigger akan dieksekusi untuk setiap baris yang diperbarui.

  • 'IF NEW.jumlah <= 10 THEN'
    adalah kondisi di mana kita akan mengirimkan pemberitahuan jika stok produk kurang dari atau sama dengan 10.

  • 'INSERT INTO notifikasi'
    adalah perintah untuk memasukkan data pemberitahuan ke dalam tabel notifikasi.

  • 'VALUES ('Stok produk dengan ID ' + NEW.id + ' hampir habis', NOW());'
    mengatur pesan pemberitahuan yang akan dimasukkan ke dalam tabel, beserta waktu pemberitahuan.

5. Enforce Aturan Bisnis :
Trigger bisa membantu Anda memastikan bahwa aturan bisnis yang penting diikuti dengan konsisten. Misalnya, Anda dapat membuat trigger yang mencegah penghapusan data pelanggan yang memiliki tagihan belum terselesaikan.

                                                
    DELIMITER //
    CREATE TRIGGER prevent_customer_deletion
    BEFORE DELETE ON pelanggan
    FOR EACH ROW
    BEGIN
        DECLARE unpaid_bill INT;

        SELECT COUNT(*) INTO unpaid_bill
        FROM tagihan
        WHERE pelanggan_id = OLD.id AND status = 'Belum Lunas';

        IF unpaid_bill > 0 THEN
            SIGNAL SQLSTATE '45000'
            SET MESSAGE_TEXT = 'Tidak dapat menghapus pelanggan dengan tagihan belum lunas';
        END IF;
    END;
    //
    DELIMITER ;

  • 'prevent_customer_deletion'
    adalah nama trigger yang Anda tentukan.

  • 'BEFORE DELETE ON pelanggan'
    menunjukkan bahwa trigger akan diaktifkan sebelum data dihapus dari tabel pelanggan.

  • 'FOR EACH ROW'
    menunjukkan bahwa trigger akan dieksekusi untuk setiap baris yang akan dihapus.

  • 'DECLARE unpaid_bill INT;'
    mendeklarasikan variabel unpaid_bill yang akan digunakan dalam trigger.

  • 'SELECT COUNT(*) INTO unpaid_bill'
    menghitung jumlah tagihan yang belum lunas untuk pelanggan yang akan dihapus.

  • 'WHERE pelanggan_id = OLD.id AND status = 'Belum Lunas';'
    adalah kondisi untuk menghitung tagihan yang belum lunas untuk pelanggan yang akan dihapus.

  • 'IF unpaid_bill > 0 THEN'
    adalah kondisi di mana trigger akan mencegah penghapusan jika ada tagihan yang belum lunas.

  • 'SIGNAL SQLSTATE '45000''
    digunakan untuk memicu pengecualian dengan kode kesalahan khusus.

  • 'SET MESSAGE_TEXT = 'Tidak dapat menghapus pelanggan dengan tagihan belum lunas';'
    menetapkan pesan kesalahan yang akan ditampilkan jika kondisi tidak terpenuhi.

Berikut adalah contoh sederhana penggunaan data trigger pada database penjualan. Di dalamnya ada tabel 'tb_produk' untuk pendataan produk atau barang, dan ada table 'produk_log' untuk mencatat setiap perubahan pada 'tb_produk'. Berikut beberapa data trigger-nya.

1. Insert Produk

Trigger ini akan dieksekusi jika terjadi even 'insert' pada tabel 'tb_produk'


    DELIMITER $$
    CREATE TRIGGER insert_produk AFTER INSERT ON tb_produk FOR EACH ROW 
    BEGIN
        INSERT INTO produk_log (keterangan, waktu, nama_produk, harga, stok) 
        VALUES('Tambah produk', NOW(), NEW.nama_produk, NEW.harga, NEW.stok);
    END $$
    DELIMITER ;

  • 'CREATE TRIGGER insert_produk'
    membuat trigger dengan nama 'insert_produk'

  • 'AFTER INSERT ON tb_produk'
    trigger dieksekusi setelah 'insert' pada tabel 'tb_produk'

  • 'FOR EACH ROW'
    trigger akan dieksekusi untuk setiap baris yang dimasukkan

  • 'INSERT INTO produk_log'
    trigger akan memasukan record baru ke dalam tabel 'produk_log'

  • 'NOW()'
    mengambil tanggal dan waktu sekarang

  • 'NEW.nama_atribut'
    mengambil value atribut yang baru dimasukan
