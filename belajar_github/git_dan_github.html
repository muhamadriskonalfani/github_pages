Apa Itu Git & GitHub

Definisi

Git
adalah sebuah software yang digunakan untuk melacak dan mengelola perubahan file yang ada di dalam folder (repositori). Dengan Git, Anda dapat mengerjakan proyek secara individu dengan lebih mudah dan efisien. Git juga memungkinkan Anda untuk bekerja berkolaborasi dengan menghubungkannya ke GitHub.

GitHub
adalah platform kolaborasi berbasis web yang digunakan oleh pengembang perangkat lunak untuk menyimpan, mengelola, dan berkolaborasi dalam proyek perangkat lunak menggunakan sistem kontrol versi Git. Ini memungkinkan pengembang untuk bekerja sama dalam pengembangan perangkat lunak, melacak perubahan kode, dan memfasilitasi proses kolaboratif.

Istilah Pada Git

Berikut adalah beberapa hal penting yang harus Anda ketahui sebelum bekerja dengan Git dan GitHub:

1. Repositori

Sebuah repositori pada Git adalah sebuah tempat penyimpanan data yang digunakan untuk mengelola proyek perangkat lunak. Repositori ini mengandung seluruh riwayat perubahan (commit) serta berkas-berkas yang terkait dengan proyek tersebut.

baca selengkapnya

Berikut langkah-langkah untuk membuat repositori di komputer Anda:

1. Membuka Git Bash / Terminal :

  • Buka Git Bash / Terminal di menu Start. Atau Anda juga bisa membukanya langsung di direktori proyek Anda.

2. Inisialisasi Repositori :

  • Jika anda membuka Git Bash / Terminal selain lewat direktori proyek Anda, maka Anda harus masuk ke direktori tersebut terlebih dahulu di dalam Git.

    cd /disk/nama_direktori

  • Jika sudah masuk ke direktori, Jalankan perintah berikut untuk menginisialisasi repositori Git :

    git init

  • Setelah Anda melakukan inisialisasi, direktori Anda akan menjadi repositori Git dan memiliki keterangan (master). (master) adalah branch atau cabang utama pada repositori Anda.

2. Commit

Commit adalah tindakan atau proses menyimpan perubahan yang telah Anda buat pada proyek Anda ke dalam repositori Git. Setiap perubahan yang Anda lakukan harus Anda simpan menggunakan commit.

baca selengkapnya

Berikut langkah-langkah menyimpan perubahan menggunakan commit:

1. Periksa Repositori Git

  • Sebelum menyimpan perubahan, disarankan untuk memeriksa apa saja yang berubah pada repositori Git, Anda bisa menjalankan perintah berikut:

    git status
2. Menyimpan Perubahan

  • Jalankan perintah ini untuk menambahkan file ke Staging Area

    git add nama_file

  • Lakukan commit dan tambahkan keterangan perubahan

    git commit -m "Pesan Commit Anda"
3. Branch

Branch adalah cabang yang ada di dalam repositori Git. Saat Anda bekerja pada repositori Git, sebenarnya Anda bekerja pada sebuah Branch, Branch tersebut dinamakan Branch utama (master). Dengan membuat Branch baru, Anda bisa melakukan perubahan pada repositori Anda tanpa mempengaruhi Branch utamanya.

baca selengkapnya

Berikut adalah beberapa perintah Branch pada repositori Git:

  • Membuat branch baru.

    git branch nama_branch

  • Berpindah ke branch tertentu.

    git checkout nama_branch

  • Membuat branch baru sekaligus berpindah ke branch tersebut.

    git checkout -b nama_branch

  • Menampilkan branch apa saja yang telah dibuat. Branch aktif ditandai dengan warna hijau.

    git branch
4. Merge

Merge adalah proses menggabungkan perubahan dari satu branch ke branch lainnya. Seperti ketika Anda telah membuat branch baru dan melakukan suatu perubahan, dan Anda ingin menerapkan perubahan tersebut ke dalam branch utamanya, maka Anda bisa melakukan merge.

Repositori

Definisi

Repositori adalah sebuah tempat penyimpanan data yang digunakan untuk mengelola proyek perangkat lunak. Repositori ini mengandung seluruh riwayat perubahan (commit) serta berkas-berkas yang terkait dengan proyek tersebut. Dalam konteks Git, repositori ini dapat diperlakukan sebagai basis data yang mencatat semua perubahan yang terjadi pada proyek, dan ini memungkinkan kolaborator proyek untuk bekerja bersama, melacak perubahan, dan mengelola versi dari berkas-berkas proyek.

Jenis Repositori

Ada 2 jenis repositori pada Git:

1. Repositori Lokal (Local Repository):
Ini adalah repositori yang berada di komputer pengguna. Pengguna dapat melakukan perubahan pada berkas-berkas proyek dan menyimpan perubahan tersebut dalam repositori lokal. Repositori lokal ini adalah tempat utama di mana pengembang bekerja pada proyek mereka.

2. Repositori Jarak Jauh (Remote Repository):
Repositori ini biasanya berada di server atau layanan hosting Git seperti GitHub, GitLab, atau Bitbucket. Repositori jarak jauh ini berfungsi sebagai tempat pusat untuk berbagi perubahan dengan pengembang lainnya. Kolaborator dapat mengambil (pull) perubahan dari repositori jarak jauh dan mengirimkan (push) perubahan mereka ke sana.

Membuat Repositori

Membuat repositori Git, baik repositori lokal maupun repositori jarak jauh (remote), melibatkan beberapa langkah dasar. Berikut adalah panduan singkat untuk membuat kedua jenis repositori tersebut:

1. Membuat Repositori Lokal:

Membuat repositori lokal dapat dilakukan dengan menginisialisasi direktori atau folder proyek di dalam Git. Berikut langkah-langkahnya.

  • Buka Terminal atau Command Prompt:
    Untuk membuat repositori Git lokal, Anda perlu membuka terminal atau command prompt di komputer Anda.

  • Pindahkan ke Direktori Proyek Anda:
    Gunakan perintah cd untuk berpindah ke direktori proyek Anda. Misalnya, jika proyek Anda ada di direktori "my_project", Anda dapat mengetikkan cd my_project untuk masuk ke dalamnya.

  • Inisialisasi Repositori Git:
    Gunakan perintah berikut untuk menginisialisasi repositori Git lokal:

    git init

    Ini akan membuat direktori .git yang menyimpan semua informasi tentang repositori Git.

2. Membuat Repositori Jarak Jauh:

Ada 2 cara untuk membuat repositori jarak jauh, Anda dapat menduplikat repositori dari GitHub ke komputer lokal Anda, Anda juga dapat membuat direktori Anda menjadi repositori di GitHub

1. Menduplikat Repositori GitHub Ke Komputer Lokal

Untuk menduplikat (clone) repositori yang ada di GitHub ke komputer lokal Anda, ikuti langkah-langkah berikut:

baca selengkapnya

  • Buka Repositori di GitHub:
    Buka halaman repositori yang ingin Anda duplikat di GitHub melalui browser Anda.

  • Dapatkan URL Repositori:
    Di halaman repositori GitHub tersebut, klik tombol 'Code' yang berwarna hijau di sebelah kanan atas. Pastikan Anda telah memilih opsi "HTTPS". Salin URL yang ada di bawah opsi tersebut.

  • Membuka Git Bash / Terminal :
    Membuka Git Bash / Terminal di komputer lokal Anda.

  • Gunakan Perintah 'git clone':
    Masuk ke direktori tempat Anda ingin menduplikat repositori. Kemudian jalankan perintah 'git clone' dengan menambahkan URL repositori yang sudah Anda salin sebelumnya.

    git clone https://github.com/username/nama-repositori.git

  • Masukkan Informasi Autentikasi (Jika Diperlukan):
    Jika repositori GitHub yang Anda duplikat bersifat pribadi atau memerlukan autentikasi, Anda mungkin diminta untuk memasukkan username dan kata sandi GitHub Anda atau menggunakan metode otentikasi lainnya.

  • Repositori Terduplikat:
    Setelah menjalankan perintah git clone, repositori GitHub tersebut akan diduplikat ke komputer lokal Anda dalam direktori yang sesuai dengan nama repositorinya.

Sekarang Anda telah berhasil menduplikat repositori dari GitHub ke komputer lokal Anda. Anda dapat bekerja dengan repositori ini secara lokal dan melakukan git push untuk mengunggah perubahan Anda kembali ke GitHub jika diperlukan.

2. Membuat Remote Dari Direktori Lokal

Untuk membuat remote repository di GitHub dari repositori yang sudah ada di komputer lokal, Anda dapat mengikuti langkah-langkah berikut:

baca selengkapnya

  • Buka GitHub:
    Buka GitHub di browser Anda. Pastikan Anda sudah login ke akun GitHub Anda.

  • Buat Repositori Baru di GitHub:
    Buat repositori baru di GitHub dengan mengklik tombol "+ New" di sudut kanan atas halaman utama GitHub Anda.

  • Isi Informasi Repositori:
    Isi informasi repositori baru, termasuk nama repositori, deskripsi, opsi visibilitas, dan opsi lainnya sesuai kebutuhan Anda. Jika Anda ingin membuat repositori kosong, pastikan opsi "Initialize this repository with a README" tidak dicentang. Klik "Create repository" untuk membuatnya.

  • Tambahkan Remote Repository:
    Setelah repositori GitHub baru Anda dibuat, Anda akan melihat petunjuk untuk menghubungkannya dengan repositori lokal yang sudah ada. Biasanya, Anda akan melihat sesuatu seperti ini:

  • Push ke Remote Repository:
    Setelah menambahkan remote, Anda dapat melakukan git push untuk mengunggah perubahan dari repositori lokal ke repositori GitHub. Contoh:

    git push -u origin master

Sekarang, repositori lokal Anda akan terhubung dengan repositori di GitHub, dan Anda dapat berkolaborasi dan berbagi kode dengan orang lain melalui GitHub. Pastikan untuk mengganti username dan nama-repositori dengan informasi yang sesuai dengan repositori GitHub Anda.

Definisi

Remote repositories adalah repositori Git yang berada di server atau lokasi lain dan biasanya digunakan untuk berkolaborasi dengan pengembang lain atau untuk menyimpan salinan cadangan dari proyek Anda.

Beberapa tugas umum yang dapat Anda lakukan dengan git remote adalah:

baca selengkapnya

    git remote -v

    Jika Anda ingin menghapus referensi ke remote repository, Anda dapat menggunakan perintah:

    git remote remove nama-remote

    Untuk memperbarui informasi tentang remote repository, misalnya jika alamat URL-nya berubah, Anda dapat menggunakan perintah:

    git remote set-url nama-remote url-baru

Git Remote memungkinkan Anda untuk berinteraksi dengan remote repository, mengambil perubahan terbaru dari remote, mengirim perubahan lokal ke remote, dan berkolaborasi dengan pengembang lain dalam pengembangan proyek. Ini adalah bagian penting dari penggunaan Git dalam proyek yang melibatkan tim atau proyek yang disimpan di server jarak jauh.

Commit

Definisi

Commit adalah tindakan atau proses menyimpan perubahan yang telah Anda buat pada proyek Anda ke dalam repositori Git. Setiap perubahan yang Anda lakukan harus Anda simpan menggunakan commit.

Ketika Anda melakukan commit, Anda menyimpan perubahan yang telah Anda buat sejak commit sebelumnya. Setiap commit memiliki pesan yang menggambarkan perubahan yang telah dilakukan, yang biasanya digunakan untuk memberikan pemahaman tentang tujuan atau konteks dari commit tersebut.

Isi Dari Commit

Commit di Git memiliki beberapa aspek penting:

  • Unik Identifier (Hash Code):
    Setiap commit memiliki identifikasi unik dalam bentuk hash yang mencerminkan isi dari commit tersebut. Ini memungkinkan Anda untuk dengan mudah merujuk ke commit tertentu.

  • Waktu dan Tanggal:
    Setiap commit mencatat waktu dan tanggal ketika commit itu dibuat.

  • Pesan Commit:
    Pesan commit adalah deskripsi singkat dari perubahan yang dilakukan dalam commit tersebut. Pesan ini harus jelas dan deskriptif untuk memudahkan pemahaman apa yang telah berubah.

Melakukan Commit

Berikut langkah-langkah menyimpan perubahan menggunakan commit:

1. Periksa Repositori Git

  • Sebelum menyimpan perubahan, disarankan untuk memeriksa apa saja yang berubah pada repositori Git, Anda bisa menjalankan perintah berikut:

    git status
2. Menyimpan Perubahan

  • Jalankan perintah ini untuk menambahkan file ke Staging Area

    git add nama_file

  • Lakukan commit dan tambahkan keterangan perubahan

    git commit -m "Pesan Commit Anda"

Branch adalah cabang yang ada di dalam repositori Git. Saat Anda bekerja pada repositori Git, sebenarnya Anda bekerja pada sebuah Branch, Branch tersebut dinamakan Branch utama (master). Dengan membuat Branch baru, Anda bisa melakukan perubahan pada repositori Anda tanpa mempengaruhi Branch utamanya.

Perintah Branch

Berikut adalah beberapa perintah Branch pada repositori Git:

  • Membuat branch baru.

    git branch nama_branch

  • Berpindah ke branch tertentu.

    git checkout nama_branch

Implementasi

Misalnya, Anda mempunyai sebuah repositori yang didalamnya ada file 'mahasiswa.html'. Anda memiliki ide untuk menambahkan file atau fitur baru pada repositori Git Anda namun Anda belum yakin untuk menambahkannya ke dalam repositori Anda. Anda bisa melakukan dengan membuat Branch baru.

1. Membuat Branch baru

  • Misalnya, Anda membuat Branch baru 'dosen':

Checkout

Definisi

adalah perintah yang digunakan untuk beralih antara branch, commit, atau tag yang berbeda dalam repository Git Anda. Ini memungkinkan Anda untuk bekerja dengan berbagai versi dari kode sumber yang ada dalam repositori Anda.

Fungsi Checkout

Checkout memiliki beberapa fungsi utama:

1. Pindah ke Branch:
Anda dapat menggunakan git checkout untuk beralih ke branch lain dalam repository Anda. Misalnya, jika Anda ingin beralih dari branch "master" ke branch "develop", Anda dapat menjalankan perintah.

git checkout develop

2. Beralih ke Commit Tertentu: Anda juga dapat menggunakan git checkout untuk beralih ke commit tertentu dalam sejarah repositori Anda. Misalnya, jika Anda ingin melihat kode pada commit dengan ID tertentu, Anda dapat menjalankan perintah git checkout < commit_id>.

3. Membuat Branch Baru:
Dengan opsi -b, git checkout memungkinkan Anda untuk membuat branch baru dan beralih ke branch tersebut sekaligus. Misalnya, git checkout -b feature-branch akan membuat branch baru dengan nama "feature-branch" dan secara otomatis beralih ke branch tersebut.

4. Beralih ke Tag:
Anda dapat menggunakan git checkout untuk beralih ke tag tertentu dalam sejarah repository. Misalnya, jika Anda ingin memeriksa kode pada saat rilis versi 1.0, Anda dapat menjalankan perintah git checkout v1.0.

5. Detached HEAD State:
Saat Anda menggunakan git checkout untuk beralih ke commit tertentu, Anda akan berada dalam mode "detached HEAD". Ini berarti Anda tidak berada di branch yang dapat diubah, tetapi hanya di commit tertentu. Ini berguna untuk melihat sejarah atau membuat perubahan eksperimental tanpa membuat branch baru.

Merge

Definisi

Merge adalah proses menggabungkan perubahan dari satu branch ke branch lainnya. Seperti ketika Anda telah membuat branch baru dan melakukan suatu perubahan, dan Anda ingin menerapkan perubahan tersebut ke dalam branch utamanya, maka Anda bisa melakukan merge.

Merge digunakan untuk menggabungkan 2 Branch. Merge dilakukan di dalam Branch utama. Perubahan yang ada pada sebuah Branch akan diterapkan pada Branch utama. Setelah melakukan Merge, Anda dapat menghapus Branch yang sudah tidak dibutuhkan.

Melakukan Merge

Berikut cara penggunaan perintah Merge pada Git :

  • Melakukan Merge atau penggabungan. 

    git merge nama_branch

  • Menampilkan Branch mana yang di-Merge

    git branch --merged

Ada 2 jenis Merge

  • Fast Forward :
    terjadi ketika Branch yang ingin kita gabungkan berada dalam jalur langsung (direct path)

  • Three way Merge :
    terjadi ketika Branch digabungkan dengan Branch yang sebelumnya digabungkan dengan Branch utama. Saat melakukan Merge, akan dibuat commit baru.

Misalnya, Dari Branch 'master' kita membuat 2 Branch baru. Saat kita melakukan Merge, Merge pertama disebut sebagai Fast Forward, karena Branch terhubung langsung dengan Branch utama. Dan yang kedua disebut Three way Merge karena Branch dihubungkan dengan Branch yang sebelumnya adalah Branch yang pertama digabungkan dengan Branch master.

Setelah Anda melakukan Merge, Anda bisa menghapus Branch yang tidak dibutuhkan.

Rebase

Definisi

Rebase adalah mekanisme yang dibuat untuk membuat grafik histori commit menjadi lebih sederhana. Rebase sangat membantu dalam pengerjaan proyek yang besar, membuat histori commit yang harusnya banyak dibuat menjadi lebih sederhana.

baca selengkapnya

Perhatikan contoh kasus dibawah ini.

"Di dalam proyek kolaborasi terdapat Branch utama (master), Selanjutnya Anda membuat Branch baru dan melakukan perubahan. Di Branch utama (master) juga terjadi perubahan (ada commit baru), misalnya ada rekan lain yang melakukan Merge. Dan jika Anda melakukan Merge juga maka akan menambah commit baru (Three way Merge) dan ini akan membuat histori commit menjadi lebih banyak."

Dari contoh diatas, Saat Anda membuat Branch baru dan melakukan perubahan, akan ada commit (dari Branch baru) yang dihubungkan dengan commit (dari Branch utama) dengan sebuah garis. Dengan menggunakan Rebase, Garis (dari Branch baru) tersebut digeser ke commit terbaru (dari Branch utama). Jadi jika Anda melakukan Merge, yang terjadi adalah Fast forward Merge dan tidak membuat commit baru.

Contoh Rebase

Berikut adalah contoh penggunaan rebase, dimulai dari membuat salinan repositori, membuat cabang, melakukan perubahan, terjadi commit baru pada cabang utama, rebase, dan seterusnya, dan diakhiri dengan push ke repositori Github

baca selengkapnya
1. Buat Salinan Repositori dan Cabang:

  • Fork repositori proyek utama di GitHub ke akun GitHub Anda. Misalnya, kami akan menggunakan repositori https://github.com/octocat/Spoon-Knife sebagai contoh.

  • Clone repositori forked ke komputer Anda:

    git clone https://github.com/YOUR_USERNAME/Spoon-Knife.git
    cd Spoon-Knife

  • Buat cabang baru, misalnya "fitur-baru," dan beralih ke cabang tersebut:

    git checkout -b fitur-baru
2. Lakukan Perubahan dan Commit:

  • Buat perubahan pada file atau tambahkan file baru ke dalam cabang "fitur-baru". Misalnya, tambahkan atau ubah file README.md.

  • Commit perubahan Anda:

    git add .
    git commit -m "Menambahkan perubahan pada README.md"
3. Terjadi Commit Baru pada Cabang Utama:

  • Selama Anda bekerja di cabang "fitur-baru," terjadi commit baru di cabang utama oleh kontributor lain. Untuk mencerminkan perubahan tersebut, Anda perlu mengambil perubahan tersebut ke dalam cabang "fitur-baru" Anda.

4. Rebase Cabang Fitur Anda:

  • Beralih kembali ke cabang utama:

    git checkout main

  • Ambil perubahan terbaru dari cabang utama:

    git pull origin main

  • Kembali ke cabang "fitur-baru" dan lakukan rebase:

    git checkout fitur-baru
    git rebase main

Ini akan menerapkan perubahan terbaru dari cabang utama ke cabang "fitur-baru" Anda. Selama proses rebase, Anda mungkin perlu menyelesaikan konflik jika ada konflik antara perubahan Anda dan perubahan dari cabang utama.

5. Push ke Repositori GitHub:

  • Setelah selesai menyelesaikan rebase dan menyelesaikan konflik jika ada, Anda dapat melakukan push perubahan ke repositori GitHub Anda:

    git push origin fitur-baru

  • Setelah push selesai, buka repositori forked Anda di GitHub dan buat Pull Request dari cabang "fitur-baru" ke cabang utama (biasanya "main") di repositori proyek utama.

Ini adalah contoh lengkap penggunaan rebase dalam situasi di mana Anda melakukan perubahan pada cabang fitur Anda dan ingin menjaga cabang Anda tetap terbaru dengan cabang utama repositori proyek utama. Pastikan untuk menggantikan URL repositori dan nama cabang dengan yang sesuai dalam contoh Anda.

Pull

Definisi

Pull pada Git adalah perintah yang digunakan untuk mengambil perubahan terbaru dari repositori jarak jauh (biasanya repositori asal, seperti GitHub atau GitLab) dan menggabungkannya ke dalam repositori lokal Anda. Tindakan ini memungkinkan Anda untuk selalu memiliki salinan terbaru dari proyek yang sedang dikerjakan, sehingga Anda dapat bekerja dengan kode yang paling mutakhir.

Melakukan Pull

Proses pull terdiri dari dua langkah utama:

  • Mengambil perubahan:
    Git akan terhubung ke repositori jarak jauh yang ditentukan (biasanya diidentifikasi dengan alamat URL) dan mengunduh perubahan terbaru dari cabang yang sesuai.

  • Menggabungkan perubahan:
    Setelah perubahan diunduh, Git akan mencoba menggabungkan perubahan tersebut ke dalam repositori lokal Anda. Jika tidak ada konflik (yaitu, tidak ada perubahan yang bertentangan antara repositori lokal dan jarak jauh), Git akan secara otomatis menggabungkan perubahan tersebut. Namun, jika ada konflik, Git akan memberi tahu Anda, dan Anda harus menyelesaikan konflik tersebut secara manual.

Contoh penggunaan perintah pull:

git pull origin master

Perintah di atas akan mengambil perubahan terbaru dari cabang "master" pada repositori jarak jauh dengan nama "origin" dan menggabungkannya dengan cabang "master" pada repositori lokal Anda.

Dengan melakukan pull secara teratur, Anda dapat tetap terhubung dengan perkembangan proyek yang sedang Anda kerjakan dan memastikan bahwa Anda selalu bekerja dengan kode yang paling mutakhir.

Push

Definisi

Git push adalah perintah dalam sistem kontrol versi Git yang digunakan untuk mengirim perubahan yang telah Anda komit secara lokal dalam repositori Anda ke repositori remote, sehingga memperbarui repositori remote dengan perubahan-perubahan tersebut. Ini memungkinkan kolaborator lainnya untuk melihat dan mengakses perubahan yang Anda buat dalam proyek yang sama. Perintah git push digunakan bersama dengan nama repositori remote dan nama cabang yang ingin Anda dorong (push). Sebagai contoh, "git push origin master" akan mengirimkan perubahan pada cabang "master" ke repositori remote yang disebut "origin".

Merge Conflict

    
                
                

                

                    
Definisi

                    

                        
Konflik merge terjadi dalam Git ketika sistem tidak dapat secara otomatis menggabungkan perubahan dari dua branch yang akan di-merge. Ini terjadi ketika dua branch yang akan digabungkan memiliki perubahan yang berdampak pada bagian yang sama dari kode atau file. Ketika Git tidak dapat memutuskan otomatis mana perubahan yang harus diterima, maka terjadi konflik merge.

                        
Ketika konflik merge terjadi, Git memberi tahu Anda tentang konflik ini dan menandai file yang terkena konflik dengan tanda konflik khusus. Anda perlu memecahkan konflik ini dengan tangan, yaitu dengan memilih perubahan mana yang akan diterima atau menggabungkannya secara manual.

                    

                

                


                
                

                

                    
Resolve Merge Conflict

                    

                        
Langkah-langkah umum untuk menyelesaikan konflik merge adalah sebagai berikut:

                        
baca selengkapnya

                        

                            
    
                                
  • Identifikasi Konflik:
     Ketika Anda melakukan Merge, Git akan memberi tahu Anda file yang memiliki konflik. Anda dapat melihat file tersebut untuk menentukan perubahan mana yang bertabrakan.
    • 
                                
  • Edit File Konflik:
     Buka file yang memiliki konflik menggunakan editor teks. Di dalamnya, Anda akan melihat tanda konflik seperti <<<<<<<, =======, dan >>>>>>>. Bagian di antara <<<<<<< dan ======= adalah perubahan dari branch Anda, sedangkan bagian antara ======= dan >>>>>>> adalah perubahan dari branch lain yang Anda coba gabungkan.
    • 
                                
  • Pilih Perubahan:
     Edit file untuk memilih perubahan mana yang akan diterima. Anda bisa memilih kode dari salah satu branch, menggabungkan keduanya, atau membuat perubahan baru.
    • 
                                
  • Hapus Tanda Konflik:
     Setelah Anda memutuskan perubahan yang akan diterima, hapus tanda konflik (seperti <<<<<<<, =======, dan >>>>>>>) dan simpan file.
    • 
                                
  • Commit Perubahan Konflik:
     Setelah Anda menyelesaikan semua konflik di semua file yang terkena, lakukan commit untuk menyimpan perubahan konflik yang sudah dipecahkan.
     
    git commit -a -m "Pesan commit Anda"
    • 
                                
  • Hapus Branch yang tidak dibutuhkan:
     Setelah Anda melakukan Merge, Branch yang anda gabungkan dengan Branch utama sudah tidak dibutuhkan lagi. Anda dapat menghapus Branch tersebut.
     
    git branch -d nama_branch
    • 
                            


                            
Konflik merge adalah bagian alami dari pengembangan kolaboratif dalam Git dan dapat terjadi ketika dua atau lebih kontributor bekerja pada bagian yang sama dari kode. Yang penting adalah menyelesaikan konflik ini dengan bijaksana untuk memastikan keberlanjutan proyek Anda.

Pull Request

    
                
                

                

                    
Definisi

                    

                        
Pull Request (PR) adalah mekanisme yang memungkinkan pengembang untuk mengusulkan perubahan kode mereka ke repositori proyek utama dan meminta pemilik repositori utama untuk memeriksa, meninjau, dan menggabungkan perubahan tersebut ke dalam proyek.

                    

                

                


                
                

                

                    
Definisi

                    

                        
Pull Request (PR) adalah fitur yang umum digunakan dalam sistem kontrol versi seperti Git, terutama pada platform kolaborasi seperti GitHub, GitLab, dan Bitbucket. Berikut adalah cara kerja umum dari sebuah Pull Request:

                        
baca selengkapnya

                        

                            

                                
1. Fork Repositori:

                                

                                    
Pengembang pertama kali membuat "fork" (salinan) repositori proyek utama ke akun GitHub atau platform Git lainnya mereka sendiri. Ini akan menghasilkan salinan independen dari repositori proyek utama yang mereka miliki dan kontrol sepenuhnya.

                                

                            


                            

                                
2. Buat Cabang:

                                

                                    
Setelah memiliki fork, pengembang membuat cabang (branch) baru dalam fork mereka. Di cabang ini, mereka melakukan perubahan dan penambahan kode yang mereka inginkan.

                                

                            


                            

                                
3. Commit Perubahan:

                                

                                    
Pengembang melakukan commit perubahan ke cabang mereka dengan mengikuti praktik pengembangan perangkat lunak yang baik, seperti memisahkan setiap fitur atau perbaikan ke dalam commit yang terpisah dan memberikan pesan commit yang informatif.

                                

                            


                            

                                
4. Buat Pull Request:

                                

                                    
Setelah perubahan kode selesai, pengembang kemudian membuka Pull Request dengan repositori proyek utama (yang mereka forked dari) untuk mengusulkan perubahan mereka. Mereka memberikan deskripsi tentang perubahan yang mereka lakukan, tujuan perubahan tersebut, dan jika perlu, tautan ke isu (issue) yang terkait.

                                

                            


                            

                                
5. Pemeriksaan dan Diskusi:

                                

                                    
Pemilik repositori proyek utama atau kontributor lain dapat melihat PR, memberikan komentar, menanyakan pertanyaan, dan memeriksa perubahan kode. Diskusi dan tinjauan ini membantu memastikan kualitas perubahan sebelum diintegrasikan ke repositori proyek utama.

                                

                            


                            

                                
6. Penggabungan (Merge):

                                

                                    
Setelah tinjauan selesai dan pemilik repositori utama atau kontributor lain merasa perubahan kode sudah siap untuk diintegrasikan, mereka dapat menggabungkan (merge) PR ke dalam repositori proyek utama. Ini menggabungkan semua perubahan kode yang ada di cabang PR ke dalam cabang utama repositori proyek utama.

                                

                            


                            

                                
7. Penyimpanan Kode Asli:

                                

                                    
Perubahan kode yang telah diintegrasikan ke repositori proyek utama sekarang akan tersimpan di sana dan menjadi bagian dari proyek tersebut.

                                

                            


                            

                                
8. Pembaruan Fork (Opsional):

                                

                                    
Pengembang yang mengirim PR dapat memutuskan untuk memperbarui fork mereka dengan perubahan yang telah diintegrasikan ke repositori proyek utama jika ada pembaruan di proyek utama yang harus diikutsertakan.

  • Akan muncul beberapa file yang belum terlacak oleh Git (untracked).
    • 
                                    
  • Gunakan perintah 'git add' untuk menambahkan file dari working tree ke Staging Area:
     
    git add .
    • 
                                    
  • Jika sudah, Lakukan commit untuk menyimpan perubahan:
     
    git commit -m "Menambahkan file index.html dan style.css"
    • 
                                

                            

                        

                        


                        
Setelah Anda melakukan inisialisasi pada direktori proyek, direktori tersebut menjadi repositori Git dan Anda dapat melakukan perubahan pada berkas-berkas didalam direktori proyek Anda. Setelah melakukan perubahan, jalankan perintah 'git add' dan 'git commit' untuk menyimpan perubahannya. Setiap perubahan akan direkam kedalam Area Histori. Anda bisa melihat alur perubahannya dengan perintah 'git log' atau melakukan graph.

                    

                

                

            

            

            
            

            

                
Latihan 2 - Git & GitHub

    
                
                

                

                    
Bekerja Dengan Git & GitHub

                    

                        
Berikutnya adalah contoh bekerja menggunakan Git & GitHub. Dengan menggunakan GitHub, Anda dapat menyimpan proyek Anda di repositori lokal Anda dan juga di repositori GitHub (remote). Berikut langkah-langkahnya.


                        

                            
1. Buka Website GitHub:

                            

                                
    
                                    
  • Buka GitHub di browser Anda. Pastikan Anda sudah login ke akun GitHub Anda.
    • 
                                

                            

                        


                        

                            
2. Buat Remote Repository

                            

                                
    
                                    
  • Buat repositori baru di GitHub dengan mengklik tombol "+ New" di sudut kanan atas halaman utama GitHub Anda.
    • 
                                    
  • Isi informasi repositori baru, termasuk nama repositori, deskripsi, opsi visibilitas. Pastikan opsi "Initialize this repository with a README" tidak dicentang. Klik "Create repository" untuk membuatnya.
    • 
                                    
  • Anda akan diantarkan ke halaman Quick Setup, Pilih bagian "buat remote repositori dari repositori lokal", Salin URL dari repositori GitHub yang Anda buat.
    • 
                                    
  • Buka Git Bash, Masuk ke repositori proyek Anda, Gunakan perintah 'git remote add' diikuti dengan nama remote (bebas) dan URL dari repositori GitHub yang Anda salin:
     
    git remote add origin https://github.com/username/nama-repositori.git
    • 
                                    
  • Dengan membuat remote repositori, repositori lokal Anda akan terhubung dengan repositori di GitHub
    • 
                                

                            

                        


                        

                            
3. Melakukan Perubahan Di Lokal

                            

                                
    
                                    
  • Setelah remote repositori dibuat, Anda bisa melakukan perubahan pada berkas-berkas proyek Anda di repositori lokal Anda.
    • 
                                    
  • Setelah selesai melakukan perubahan, Buka Git Bash kembali dan cek perubahan yang terjadi.
     
    git status
    • 
                                    
  • Selanjutnya tambahkan perubahan tersebut ke Staging Area
     
    git add .
    • 
                                    
  • Selanjutnya lakukan commit untuk menyimpan perubahan.
     
    git commit -m "Pesan Commit Anda"
    • 
                                

                            

                        


                        

                            
4. Melakukan Push Ke Repositori GitHub

                            

                                
    
                                    
  • Sebelumnya ketika Anda melakukan commit, perubahan yang Anda lakukan hanya disimpan di dalam repositori lokal saja. Untuk menyimpan perubahan ke repositori GitHub, Anda harus melakukan Push.
    • 
                                    
  • Gunakan perintah 'git push' untuk menyimpan perubahan dari repositori lokal ke repositori GitHub:
     
    git push -u origin master
    • 
                                

                            

                        

                        


                        
Sekarang, Semua berkas-berkas proyek yang ada di repositori lokal Anda juga ada di repositori GitHub. Ini memungkinkan Anda untuk bekerja berkolaborasi, ataupun bekerja dengan perangkat yang berbeda. Disarankan untuk melakukan 'git status' atau graph untuk melihat apakah repositori lokal Anda sudah Up To Date dengan repositori GitHub.

                    

                

                

            

            


            

            

                
Latihan 3 - Branch & Merge

    
                
                

                

                    
Branch & Merge

                    

                        
Contoh kasus, Misalkan Anda mempunyai sebuah repositori yang didalamnya ada file 'mahasiswa.html'. Anda memiliki ide untuk menambahkan file atau fitur baru pada repositori Git Anda namun Anda belum yakin untuk menambahkannya ke dalam repositori Anda. Anda bisa membuat branch baru dan melakukan perubahan tersebut. Berikut langkah-langkahnya.


                        

                            
1. Membuat Branch baru

                            

                                
    
                                    
  • Misalnya, Anda membuat Branch baru 'dosen':
     
    git branch dosen
    • 
                                    
  • Selanjutnya, Anda bisa gunakan perintah 'git branch' untuk melihat semua Branch yang telah Anda buat:
     
    git branch
    • 
                                

                            

                        


                        

                            
2. Masuk ke Branch 'dosen'

                            

                                
    
                                    
  • Sebelumnya Anda berada pada Branch 'master', Jalankan perintah ini untuk berganti ke Branch 'dosen':
     
    git checkout dosen
    • 
                                    
  • Selanjutnya, Anda bisa melakukan perubahan seperti menambahkan file baru 'dosen.html':
     
    git add dosen.html
     
    git commit -m "Menambahkan file dosen.html"
    • 
                                

                            

                        


                        

                            
3. Merge Branch 'dosen' ke 'master'

                            

                                
    
                                    
  • Pastikan Anda yakin untuk menambahkan file atau fitur baru dari Branch 'dosen' ke Branch utama 'master'
    • 
                                    
  • Jika sudah, Pertama pindah ke Branch utama 'master':
     
    git checkout master
    • 
                                    
  • Selanjutnya, Anda bisa melakukan Merge Branch 'dosen' dengan 'master':
     
    git merge dosen
    • 
                                    
  • File 'dosen.html' berhasil ditambahkan ke Branch utama 'master', Jadi di dalam Branch utama ada dua file yaitu 'mahasiswa.html' dan 'dosen.html.
    • 
                                    
  • Selanjutnya, karena Branch 'dosen' sudah di-Merge dengan Branch 'master', Anda bisa menghapus Branch 'dosen':
     
    git branch -d dosen
    • 
                                

                            

                        

                        


                        
Branch dan Merge adalah dua konsep penting dalam Git. Ini memungkinkan Anda membuat cabang baru dan melakukan perubahan pada berkas-berkas proyek Anda tanpa mempengaruhi berkas-berkas aslinya. Setelah perubahan selesai dan Anda ingin menerapkan perubahannya kedalam berkas-berkas asli, Anda bisa melakukan Merge. Setelah melakukan Merge, Anda bisa menghapus branch jika tidak dibutuhkan.

                    

                

                

            

            


            

            

                
Latihan 4 - Multiple Remote

    
                
                

                

                    
Multiple Remote

                    

                        
Contoh kasus, Misalkan Anda mengerjakan proyek secara kolaborasi menggunakan GitHub. Maka Anda akan membutuhkan 2 remote didalam Git lokal Anda, yaitu remote dari repositori GitHub Anda sendiri dan dari repositori GitHub utama. Berikut langkah selengkapnya:


                        

                            
1. Salin Repositori Utama Ke Repositori GitHub Anda:

                            

                                
    
                                    
  • Buka repositori GitHub utama dan klik tombol "Fork" untuk menyalin.
    • 
                                    
  • Buka salinan repositori, salin link repositori dan clone ke Git lokal Anda.
     
    git clone https://github.com/username/nama_repositori.git
    • 
                                

                            

                        


                        

                            
2. Buat Branch Dan Lakukan Perubahan:

                            

                                
    
                                    
  • Buka Git dan masuk ke repositori Git
    • 
                                    
  • Disarankan untuk tidak melakukan perubahan pada branch utama, Anda bisa membuat branch baru dan melakukan perubahan disana.
     
    git checkout -b features
    • 
                                    
  • Buat perubahan, simpan perubahan dan lakukan commit.
    • 
                                    
  • Push perubahan ke repositori GitHub dan lakukan pull request ke repositori utama.
    • 
                                

                            

                        


                        
Dengan melakukan pull request, Anda meminta penanggung jawab repositori utama untuk menerapkan perubahan yang Anda buat ke repositori utama. Setelah perubahan diterapkan, anda bisa men-singkronkan repositori utama dengan repositori GitHub Anda dan repositori lokal supaya selalu Up To Date. Berikut langkah selengkapnya:


                        

                            
1. Tambahkan Remote Repositori Utama:

                            

                                
    
                                    
  • Buka repositori GitHub utama dan salin link repositori.
    • 
                                    
  • Jalankan perintah berikut untuk membuat remote baru. Beri nama remote (bebas) diikuti dengan link repositori utama yang Anda salin.
     
    git remote add upstream https://github.com/username/nama_repositori.git
    • 
                                

                            

                        


                        

                            
2. Ambil Perubahan Dari Repositori Utama:

                            

                                
    
                                    
  • Pastikan Anda berada dalam branch master di repositori lokal Anda.
    • 
                                    
  • Lacak perubahan dari repositori GitHub utama dengan perintah 'git fetch' dan melakukan graph.
     
    git fetch upstream
    • 
                                    
  • Salin perubahan dengan melakukan merge dari branch master di repositori lokal Anda dengan dengan branch master dari repositori utama.
     
    git merge upstream/master
    • 
                                    
  • Sekarang repositori lokal Anda sudah "up to date" dengan repositori utama.
    • 
                                

                            

                        


                        

                            
3. Push Perubahan Ke Repositori GitHub

                            

                                
    
                                    
  • Anda bisa melakukan push untuk mengirimkan perubahan dari repositori lokal ke repositori GitHub Anda.
     
    git push -u origin master
    • 
                                    
  • Sekarang repositori GitHub Anda sudah "up to date" dengan repositori utama.
    • 
                                

                            

                        


                        
Sekarang, Anda memiliki repositori forked di akun GitHub Anda dan juga di komputer lokal Anda. Anda dapat melakukan perubahan, menjalankan commit, dan mengirim perubahan kembali ke repositori forked di akun GitHub Anda dengan perintah 'git push'.


                        
Ketika bekerja kolaborasi menggunakan GitHub, Disarankan repositori lokal dan repositori GitHub Anda selalu "up to date" dengan repositori GitHub utama.


                        
Jika Anda ingin mengambil perubahan dari repositori asli (upstream), Anda dapat menggunakan perintah 'git pull upstream main' (dengan asumsi Anda menggunakan cabang "main" sebagai cabang utama). Pastikan Anda berada dalam cabang yang benar sebelum menjalankan perintah ini.
